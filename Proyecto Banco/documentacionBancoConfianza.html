1 Introducción al sistema bancario BancoConanza
BancoConanza es un sistema bancario educativo desarrollado para la gestión de cuentas
de ahorro y corriente, utilizando estructuras de datos avanzadas y técnicas modernas de
programación orientada a objetos. El sistema automatiza la apertura de cuentas, manejo
de transacciones (depósitos y retiros), consultas avanzadas, respaldo y restauración de
datos, integrando validaciones robustas y seguridad en el manejo de la información.
1.1 Visión del sistema
BancoConanza busca ofrecer una plataforma segura, eciente y didáctica para la ad-
ministración de cuentas bancarias, enfocada en el aprendizaje y aplicación de concep-
tos avanzados de programación como listas doblemente enlazadas circulares, manejo de
memoria dinámica, funciones lambda, recursividad, plantillas (templates), sobrecarga de
métodos y funciones, así como el uso de técnicas de cifrado y respaldo de información.
1.2 Aspectos clave
• Estructura de datos: Uso exclusivo de listas doblemente enlazadas circulares
para almacenar clientes, cuentas y transacciones. No se utilizan arreglos ni librerías
externas.
• Tipos de cuentas: Soporte para cuentas de ahorro y corriente, con operaciones
automatizadas.
• Transacciones: Realización de depósitos y retiros, con actualización automática
de saldos.
• Validaciones: Implementación obligatoria de validaciones para datos críticos (cé-
dula, fechas, montos, etc.) usando bloques try-catch para manejo de excepciones.
• Consultas avanzadas: Permite consultar transacciones por rango de fechas, cri-
terios personalizados y consultas denidas por el usuario.
• Respaldo y restauración: Generación de backups en archivos binarios con nom-
bre basado en la fecha y hora, sin sobrescribir respaldos anteriores. Restauración
de cualquier backup disponible.
• Cifrado: Opciones para cifrar y descifrar archivos binarios de respaldo, garanti-
zando la condencialidad de la información.
• Programación orientada a objetos: Implementación de clases para clientes,
cuentas y transacciones, aplicando conceptos de herencia, polimorsmo, sobrecarga
y plantillas.
• Funciones lambda y recursividad: Uso de funciones lambda para operaciones
sobre listas y recursividad en búsquedas o recorridos.
• Interfaz de usuario: Interfaz de consola clara y validada para interacción con el
usuario.

1.3 Requisitos del sistema
Hardware:
• Procesador: Intel i3 o superior
• Memoria RAM: 2 GB mínimo
• Espacio en disco: 50 MB libres
Software:
• Sistema operativo: Windows, Linux o MacOS
• Compilador C++ (recomendado: GCC 9.0+, MinGW, o equivalente)
• Editor de texto o IDE compatible con C++
Dependencias:
• No requiere librerías externas; todo el código es estándar C++.
1.4 Guía rápida para el usuario
1. Apertura de cuenta:
• Seleccione el tipo de cuenta (ahorro o corriente).
• Ingrese sus datos personales (nombre, cédula, fecha de nacimiento, etc.).
• El sistema validará los datos y creará la cuenta en la lista doblemente enlazada
circular.
2. Realización de transacciones:
• Acceda a su cuenta mediante su número de cuenta o cédula.
• Seleccione la operación: depósito o retiro.
• Ingrese el monto y conrme la transacción.
• El sistema actualizará el saldo y almacenará la transacción en la lista.
3. Consultas:
• Puede consultar transacciones por rango de fechas, por parámetros person-
alizados (ej. monto, tipo de transacción) o mediante criterios avanzados
denidos por usted.
• Use el menú de consultas para seleccionar el tipo deseado.
4. Backup y restauración:
• Para crear un respaldo, seleccione la opción de backup. El archivo binario se
generará automáticamente con la fecha y hora actual.
• Para restaurar, elija el archivo de respaldo deseado y conrme la restauración.
• Puede cifrar o descifrar los archivos de backup desde el menú correspondiente.

5. Seguridad y validaciones:
• Todos los datos ingresados son validados rigurosamente.
• El sistema maneja errores y excepciones para evitar datos corruptos o incon-
sistentes.
