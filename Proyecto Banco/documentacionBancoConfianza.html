1 Introduccin al sistema bancario BancoConanza
BancoConanza es un sistema bancario educativo desarrollado para la gestin de cuentas
de ahorro y corriente, utilizando estructuras de datos avanzadas y tcnicas modernas de
programacin orientada a objetos. El sistema automatiza la apertura de cuentas, manejo
de transacciones (depsitos y retiros), consultas avanzadas, respaldo y restauracin de
datos, integrando validaciones robustas y seguridad en el manejo de la informacin.
1.1 Visin del sistema
BancoConanza busca ofrecer una plataforma segura, eciente y didctica para la ad-
ministracin de cuentas bancarias, enfocada en el aprendizaje y aplicacin de concep-
tos avanzados de programacin como listas doblemente enlazadas circulares, manejo de
memoria dinmica, funciones lambda, recursividad, plantillas (templates), sobrecarga de
mtodos y funciones, as como el uso de tcnicas de cifrado y respaldo de informacin.
1.2 Aspectos clave
Estructura de datos: Uso exclusivo de listas doblemente enlazadas circulares
para almacenar clientes, cuentas y transacciones. No se utilizan arreglos ni libreras
externas.
Tipos de cuentas: Soporte para cuentas de ahorro y corriente, con operaciones
automatizadas.
Transacciones: Realizacin de depsitos y retiros, con actualizacin automtica
de saldos.
Validaciones: Implementacin obligatoria de validaciones para datos crticos (c-
dula, fechas, montos, etc.) usando bloques try-catch para manejo de excepciones.
Consultas avanzadas: Permite consultar transacciones por rango de fechas, cri-
terios personalizados y consultas denidas por el usuario.
Respaldo y restauracin: Generacin de backups en archivos binarios con nom-
bre basado en la fecha y hora, sin sobrescribir respaldos anteriores. Restauracin
de cualquier backup disponible.
Cifrado: Opciones para cifrar y descifrar archivos binarios de respaldo, garanti-
zando la condencialidad de la informacin.
Programacin orientada a objetos: Implementacin de clases para clientes,
cuentas y transacciones, aplicando conceptos de herencia, polimorsmo, sobrecarga
y plantillas.
Funciones lambda y recursividad: Uso de funciones lambda para operaciones
sobre listas y recursividad en bsquedas o recorridos.
Interfaz de usuario: Interfaz de consola clara y validada para interaccin con el
usuario.

1.4 Gua rpida para el usuario
1. Apertura de cuenta:
Seleccione el tipo de cuenta (ahorro o corriente).
Ingrese sus datos personales (nombre, cdula, fecha de nacimiento, etc.).
El sistema validar los datos y crear la cuenta en la lista doblemente enlazada
circular.
2. Realizacin de transacciones:
Acceda a su cuenta mediante su nmero de cuenta o cdula.
Seleccione la operacin: depsito o retiro.
Ingrese el monto y conrme la transaccin.
El sistema actualizar el saldo y almacenar la transaccin en la lista.
3. Consultas:
Puede consultar transacciones por rango de fechas, por parmetros person-
alizados (ej. monto, tipo de transaccin) o mediante criterios avanzados
denidos por usted.
Use el men de consultas para seleccionar el tipo deseado.
4. Backup y restauracin:
Para crear un respaldo, seleccione la opcin de backup. El archivo binario se
generar automticamente con la fecha y hora actual.
Para restaurar, elija el archivo de respaldo deseado y conrme la restauracin.
Puede cifrar o descifrar los archivos de backup desde el men correspondiente.
